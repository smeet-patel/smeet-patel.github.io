Planning

Project Summary

This project is an application development for a (fictitious) burger restaurant in Wellington which aims to allow customers to create a custom burger in a user-friendly manner. We will be implementing a database to store the orders, burger ingredients, and a list of customers connected with the production line application at Dimmu Burger location. The completion of this project would benefit Dimmu Burger and its users because of a user-friendly interface that allows customer to order, and hence improve the number of customer orders and customer experience.

Project Deliverables

  1. Design and planning report (Due Tuesday 6th November 2018)
  2. Live demonstration (To be presented on Thursday 15th Nov)
  3. Final report (Submit by Sunday 18th November)
  • This was a 2-week project and a week consist of 4 days.

App Stakeholders

  • Customers order burgers by selecting ingredients on the web application.
  • Admin / senior member of kitchen can Check ingredient stock levels and Order new ingredients as needed.
  • Kitchen staff will receive new orders and Change status of orders (received, in progress, complete).

Project Timeline

Requirements

Web App

  • Customers
    1. Can check for ingredient availability - only show ingredients available
    2. Can create custom burger(s)
    3. Can select multiple of the same ingredients
    4. Can order burger
    5. Can order multiple burgers
    6. Can be notified of progress of order
    7. The user interface should be responsive, but the production interface is not responsive.

Production Line App

  • Kitchen Staff(s)
    1. Can receive customer order
    2. Can check if ingredients are available
    3. Can mark off orders as made
  • Admin Staff(s)
    1. Can check ingredient stock levels
    2. Can order new ingredients as require
    3. Require login

Database

  • Ingredients
    • Ingredient category (e.g.: Bread, Meat/Vege, Produce, Cheese)
    • Specific ingredient name (e.g. Chicken, Beef, Falafel)
    • Quantity in stock
    • Minimum restock level
  • Burger recipes
    • Order number
    • Burgers required (may be multiple)
    • Order progress
  • Logins
    • Login details for Production line admin staff

Application Wireframes

I often use wireframes for projects which have user interfaces as it helps me stay focused when developing the application or prototype because they are simple outlines of the UI layout. Many of the elements are placeholders, so this allows me to determine if I need to acquire or create content, develop the visuals and functionality.

The first wireframe is the landing page of the web application with a banner on top and branding statement below.
The second wireframe shows the customer ordering form with a step progress bar on top to show the customer their current status.

The third wireframe shows how the customer can complete and submit the order by validation of email and user details.
The fourth wireframe displays the login screen for the Java production line application, with a simple entry for the kitchen staff and a login for the admin staff.
