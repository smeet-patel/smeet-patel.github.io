I wanted to explore python and C# programming languages by creating two almost identical console base banking system applications with simple features such as adding a new customer and account, deleting a records, modifying details, and allowing bankers to look the records of all customers. this allowed for transferring payments from one customer to another, clearing and accepting a cheque. Also, for python, I made a real-time currency conversion using APIs.

The design of the application was designed to allow the user to input a number which corresponds to a certain user for example 1 for ‘new customer’, 2 for ‘existing customer’ and 3 for ‘banker’. Depending on the selected input, the user would get other options to complete the task as mentioned above. If the user was a new user, they are required to input their personal details to create their bank account.

The reason I choose Python and C# programming languages was due to the popularity of the languages in the IT industry and this is evident in a recent article related to GitHub usage (they are 3rd and 6th most used languages respectively).

Python

  • Overall, python was more natural to learn and was unique as it was different to JAVA or C# due to had no curly braces, therefore correctly indenting the code was crucial.
  • Python was easier to implement the APIs in JSON format and works better with most data types.
  • I learnt that I had to cast numeric values input into int or float when storing or programming mathematical operation. For example, 2+1 would be 21 instead of equalling to 3 while using string type.
  • One of the challenges in python was trying to access a global variable inside a function. To solve the issues, I had to insert ‘global’ before the variable inside the function.
  • For my data structure, I used a nested dictionary because it enabled me to set up a data structure like a HashMap in Java and therefore it allowed me to store the customer number ID as key and the other information for that customer number along aside it.
  • I was also able to implement a real time currency conversion calculator from one currency to another by accessing a JSON file online. This was possibly my biggest challenge in python as I had to convert retrieving the exchange rate and program the operations.
  • Overall, I really enjoyed programming in python and in the future, I would use Python for machine learning projects and participate in Kaggle competitions.
